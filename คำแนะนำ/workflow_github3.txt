## 1️⃣ การตรวจสอบอะไรเพิ่มได้อีกบ้าง

### 🟡 ที่ควรเพิ่ม (แนะนำสูง)

| # | Check | ทำไมต้องมี | วิธีทำ |
|---|-------|------------|--------|

## 2️⃣ ไฟล์ .yml ทำออกมาดีมั้ย / ปรับแต่งได้มั้ย

**สรุปสั้น ๆ: โดยรวมดีมาก (คุณใส่ใจมาก — เห็นได้จาก comments ละเอียด, fix tags, retry logic) แต่มีจุดที่ควรปรับ:**

### ✅ จุดแข็งที่ดีอยู่แล้ว
- ใช้ pinned SHA สำหรับ third-party actions (`softprops/action-gh-release@718ea10...`) — ผ่าน SonarCloud S7637
- มี concurrency group + cancel-in-progress ที่ถูกต้อง
- `02-deploy.yml` มี retry + diagnostic ที่ robust มาก (3 วิธีในการดึง access_token)
- `07-doc-code-sync.yml` มี critical/warning tier แยกชัดเจน
- ใช้ environments (`production`) ใน deploy

### ⚠️ จุดที่ควรปรับปรุง

**Bug/Inconsistency ใน `05-scheduled-health.yml`:**
```bash
# ❌ Bug — บอก 22 แต่ README/CI อื่น ๆ บอก 25
if [[ "$total_files" -eq 22 ]]; then
    echo "  ✅ ไฟล์ครบ 22 (00-21)"
```
แก้เป็น 25 และใช้ range แบบ dynamic

**`04-release.yml` — auto-bump patch ทุก push:**
```yaml
# ⚠️ ปัจจุบัน: push → main → src/** → bump patch → release
# ปัญหา: PR merge ทุกครั้งจะสร้าง release ใหม่ → spam
```
แนะนำ: ใช้ conventional commits → determine bump (feat=minor, fix=patch, BREAKING=major) ผ่าน `action-semantic-release` หรือเขียน custom

**`06-codeql.yml` — copy .gs → .js แต่ path mapping หาย:**
```yaml
# ⚠️ CodeQL จะรายงาน finding ที่ path "src/foo.js" แต่ของจริงคือ "src/foo.gs"
# → Developer ต้องค้นหาเอง
```
แก้: ใช้ `paths-ignore` + filter results ใน SARIF, หรือ symlink แทน copy

**`01-ci.yml` — syntax check แบบ naive:**
```bash
# ⚠️ paren/brace count แบบนี้ false-positive ง่าย:
#   - String ที่มี ")" หรือ "}" → นับผิด
#   - regex /[)]/ → นับผิด
#   - comment // } → นับผิด
```
แก้: ใช้ Node.js `acorn.parse()` หรือ Babel parser แทน

**`02-deploy.yml` — verify job เปลือง quota:**
- pull ทั้งโปรเจกต์กลับมา diff ทุกครั้ง → ช้า + กิน API quota (60 req/min)
- ควร trust clasp push เมื่อ HTTP 200 และ verify แค่ file count / version number

### 💡 แนะนำเพิ่มเติม

1. **Permissions ต่อ job ให้แคบลง** — ตอนนี้หลาย job ไม่ได้ระบุ permissions เลย ใช้ default ที่กว้างเกินไป
2. **Pin GitHub Action versions ด้วย SHA ทั้งหมด** — บางที่ใช้ `@v7`, `@v6` ซึ่งเปลี่ยนได้
3. **Reusable workflow** — แยก deploy/dev/deps-check ออกเป็น composite action เพื่อ reuse
4. **Caching** — `npm ci` ทุกครั้ง ควร cache `.npm` directory

#!/usr/bin/env bash
# Check 6 — Header Structure Completeness
# ตรวจว่า header ของทุกไฟล์ .gs มี section ครบ 6 ส่วน:
#   1. VERSION
#   2. FILE
#   3. PURPOSE       (non-empty)
#   4. CHANGELOG     (mentions /docs/CHANGELOG.md)
#   5. DEPENDENCIES  (REQUIRES + CALLED BY / CALLS)
#   6. ARCHITECTURE  (non-empty)
#
# ไฟล์ที่ขาด section ใด section หนึ่ง → fail
#
# Returns:
#   0 = pass (all files have complete header)
#   1 = fail (some files have missing/empty sections)

set -uo pipefail
cd "$(dirname "$0")/../../.."

echo "📋 Check 6: Header Structure Completeness"

failures=0
warnings=0
checked=0

# Required sections (each must appear AND have non-empty content)
required_sections=("VERSION" "FILE" "PURPOSE" "CHANGELOG" "DEPENDENCIES" "ARCHITECTURE")

# DEPENDENCIES must contain at least one of: REQUIRES, CALLED BY, CALLS
required_dep_subsections=("REQUIRES")

for f in $(find src -name "*.gs" -type f | sort); do
  checked=$((checked+1))

  # Extract header block: from /** to first */
  header=$(awk '/^\/\*\*$/,/^\*\/$/' "$f")
  if [[ -z "$header" ]]; then
    echo "  ❌ $f: ไม่พบ header block (/** ... */)"
    failures=$((failures+1))
    continue
  fi

  file_failures=0
  file_warnings=0

  for section in "${required_sections[@]}"; do
    # Look for "* <SECTION>:" or "* <SECTION> " in header
    if ! echo "$header" | grep -qE "^\s*\*\s*${section}\s*[:[:space:]]"; then
      echo "  ❌ $f: ขาด section [$section]"
      file_failures=$((file_failures+1))
    fi
  done

  # PURPOSE: must have non-empty content
  purpose_content=$(echo "$header" | awk '/^\s*\*\s*PURPOSE\s*:/{flag=1; next} /^\s*\*\s*={3,}/ || /^\s*\*\s*[A-Z][A-Z_]+:/ {if (flag && !/PURPOSE/) {flag=0}} flag' | sed 's/^\s*\*\s*//' | grep -vE "^\s*$" | head -3)
  if [[ -z "$purpose_content" ]]; then
    echo "  ⚠️  $f: PURPOSE section ว่างเปล่า"
    file_warnings=$((file_warnings+1))
  fi

  # ARCHITECTURE: must have non-empty content
  arch_content=$(echo "$header" | awk '/^\s*\*\s*ARCHITECTURE\s*:/{flag=1; next} /^\s*\*\s*={3,}/ || /^\s*\*\s*[A-Z][A-Z_]+:/ {if (flag && !/ARCHITECTURE/) {flag=0}} flag' | sed 's/^\s*\*\s*//' | grep -vE "^\s*$" | head -3)
  if [[ -z "$arch_content" ]]; then
    echo "  ⚠️  $f: ARCHITECTURE section ว่างเปล่า"

#!/usr/bin/env bash
# Check 7 — REQUIRES / CALLS Validation
# Parse DEPENDENCIES section แล้ว verify:
#   1. REQUIRES — ไฟล์ที่อ้างถึงต้องมีอยู่จริง
#   2. CALLS / CALLED BY — function ที่อ้างถึงต้องมี definition ในไฟล์เป้าหมาย
#
# Format ที่ parse ได้:
#   - 01_Config.gs     (description)              ← REQUIRES
#   - runMatchEngine() → 10_MatchEngine.gs       ← CALLS
#   - 06_PersonService (normalizePersonNameFull) ← EXPORTS TO
#
# Returns:
#   0 = pass (no phantom refs)
#   1 = fail (some refs don't resolve)

set -u
cd "$(dirname "$0")/../../.."

echo "📋 Check 7: REQUIRES / CALLS Validation"

failures=0
warnings=0
checked=0

# Build map of: file -> set of function names defined (for fast lookup)
echo "  🔧 Building function index..."
declare -A FUNC_MAP

for f in $(find src -name "*.gs" -type f); do
  # Extract top-level and nested function definitions
  # Match: function name(...) or name: function(...) or async function name(...)
  funcs=$(grep -oE "(^|[[:space:]])(function[[:space:]]+[a-zA-Z_\$][a-zA-Z0-9_\$]*|[a-zA-Z_\$][a-zA-Z0-9_\$]*[[:space:]]*[:=][[:space:]]*function)" "$f" 2>/dev/null \
    | grep -oE "[a-zA-Z_\$][a-zA-Z0-9_\$]*[[:space:]]*[:=]?[[:space:]]*function|[a-zA-Z_\$][a-zA-Z0-9_\$]*" \
    | grep -oE "^[a-zA-Z_\$][a-zA-Z0-9_\$]*$" | sort -u)
  FUNC_MAP["$f"]="$funcs"
done

echo "  📊 Indexed ${#FUNC_MAP[@]} files"

# Function: list all function names defined in a file
list_funcs() {
  local target="$1"
  # Try with .gs appended
  if [[ -n "${FUNC_MAP[$target]:-}" ]]; then
    echo "${FUNC_MAP[$target]}"
    return
  fi
  # Try matching by basename
  local base=$(basename "$target" .gs)
  for k in "${!FUNC_MAP[@]}"; do
    if [[ "$(basename "$k" .gs)" == "$base" ]]; then
      echo "${FUNC_MAP[$k]}"
      return
    fi
  done
}

# Function: check if a function name exists in a file
func_exists() {
  local func="$1"
  local file="$2"
  if [[ -z "$func" || -z "$file" ]]; then
    return 1
  fi
  # Try direct lookup
  if [[ -n "${FUNC_MAP[$file]:-}" ]]; then
    grep -qxF "$func" <<< "${FUNC_MAP[$file]}" && return 0
  fi
  # Try by basename
  local base=$(basename "$file" .gs)
  for k in "${!FUNC_MAP[@]}"; do
    if [[ "$(basename "$k" .gs)" == "$base" && -n "${FUNC_MAP[$k]:-}" ]]; then
      grep -qxF "$func" <<< "${FUNC_MAP[$k]}" && return 0
    fi
  done
  return 1
}

# Function: check if a file path exists
file_exists_in_repo() {
  local target="$1"
  # Strip group annotation like "(Group 0)"
  target=$(echo "$target" | sed -E 's/[[:space:]]*\(Group[[:space:]]+[0-9]+\)//')
  target=$(echo "$target" | sed -E 's/[[:space:]]*\(self\)//')
  target=$(echo "$target" | awk '{print $1}')

  # Try direct
  [[ -f "src/$target" ]] && return 0
  [[ -f "src/O_core_system/$target" ]] && return 0
  [[ -f "src/1_group1_master_db/$target" ]] && return 0
  [[ -f "src/2_group2_daily_ops/$target" ]] && return 0
  [[ -f "src/3_group3_webapp/$target" ]] && return 0
  [[ -f "src/4_group4_pipeline_mgr/$target" ]] && return 0
  return 1
}

# Function: resolve file path to full path
resolve_file() {
  local target="$1"
  target=$(echo "$target" | sed -E 's/[[:space:]]*\(Group[[:space:]]+[0-9]+\)//')
  target=$(echo "$target" | sed -E 's/[[:space:]]*\(self\)//')

    fi
  done
  echo ""
}

    # Format 2: "TargetFile (func1, func2)"
    # Format 3: "func1()  →  TargetFile.gs"

    if [[ "$line" == *"→"* ]]; then

      funcs_part=$(echo "$line" | awk -F'→' '{print $1}')

      target_path=$(resolve_file "$target_part")

        continue
      fi

# Check 8 — CHANGELOG Sync (header ↔ /docs/CHANGELOG.md)
#
# ตรวจ 3 อย่าง:
#   1. APP_VERSION (จาก 01_Config.gs) ต้องปรากฏใน /docs/CHANGELOG.md
#   2. Header VERSION: ของทุกไฟล์ .gs ต้องตรงกับ APP_VERSION
#      (check_01 ทำอยู่แล้ว — เรา check เพิ่มว่า version นั้นต้องมีใน CHANGELOG)
#   3. ถ้า CHANGELOG.md มี version ใหม่กว่า APP_VERSION → warn
#      (อาจมีคนเพิ่ม CHANGELOG ก่อน bump code)
#
# Returns:
#   0 = pass (versions in sync)
#   1 = fail (CHANGELOG.md doesn't know about current APP_VERSION)

set -u

echo "📋 Check 8: CHANGELOG Sync (header ↔ /docs/CHANGELOG.md)"

if [[ ! -f "docs/CHANGELOG.md" ]]; then
  echo "  ❌ ไม่พบ docs/CHANGELOG.md"
  exit 1

# Extract APP_VERSION

if [[ -z "$APP_VERSION" ]]; then
  echo "  ❌ ไม่พบ APP_VERSION ใน 01_Config.gs"
  exit 1
fi
echo "  📋 APP_VERSION: $APP_VERSION"

failures=0
warnings=0

echo ""
echo "  📌 Check: APP_VERSION ต้องปรากฏใน /docs/CHANGELOG.md"

# Match patterns like "| 6.0.036 |", "## [6.0.036]", "### 6.0.036", etc.
if grep -qE "(\|[[:space:]]*${APP_VERSION//./\\.}[[:space:]]*\|)|^#+[[:space:]]+${APP_VERSION//./\\.}\b" docs/CHANGELOG.md; then
  echo "    ✅ ${APP_VERSION} พบใน /docs/CHANGELOG.md"
else
  echo "    ❌ ${APP_VERSION} ไม่พบใน /docs/CHANGELOG.md"
  echo "       → เพิ่ม row ใน Versions Summary table:"
  echo "         | ${APP_VERSION} | $(date -u '+%Y-%m-%d') | <CYCLE_NAME> | <description> |"
  failures=$((failures+1))
fi

# Check 2: Find the latest version mentioned in CHANGELOG.md
echo ""
echo "  📌 Check: CHANGELOG.md vs APP_VERSION consistency"

# Get latest version from CHANGELOG (first occurrence in Versions Summary table)
latest_in_changelog=$(grep -oE "\|[[:space:]]*[0-9]+\.[0-9]+\.[0-9]+[[:space:]]*\|" docs/CHANGELOG.md | head -1 | grep -oE "[0-9]+\.[0-9]+\.[0-9]+" || echo "")

if [[ -n "$latest_in_changelog" ]]; then
  echo "    📋 Latest version in CHANGELOG.md: $latest_in_changelog"

  # Compare versions (use sort -V)
  if [[ "$latest_in_changelog" != "$APP_VERSION" ]]; then
    # Determine which is greater
    higher=$(printf "%s\n%s\n" "$latest_in_changelog" "$APP_VERSION" | sort -V | tail -1)
    if [[ "$higher" == "$APP_VERSION" ]]; then
      echo "    ⚠️  APP_VERSION ($APP_VERSION) > CHANGELOG latest ($latest_in_changelog)"
      echo "       → น่าจะ bump code ก่อนอัปเดต CHANGELOG (อัปเดต CHANGELOG ด้วย)"
      warnings=$((warnings+1))
    else
      echo "    ⚠️  CHANGELOG has newer version ($latest_in_changelog) than APP_VERSION ($APP_VERSION)"
      echo "       → มีคนเพิ่ม CHANGELOG entry แต่ยังไม่ bump code (อัปเดต APP_VERSION)"
      warnings=$((warnings+1))
    fi
  else
    echo "    ✅ APP_VERSION matches CHANGELOG latest"
  fi
fi

# Check 3: All .gs header VERSION must be in CHANGELOG.md

header_versions=$(grep -rhE "^\s*\*\s*VERSION:\s*[0-9]+\.[0-9]+\.[0-9]+" src/ 2>/dev/null \

for ver in $header_versions; do
  # Skip if it's the current APP_VERSION (already checked above)
  [[ "$ver" == "$APP_VERSION" ]] && continue

  # For older versions, just verify they appear somewhere in CHANGELOG
  if ! grep -qE "(\|[[:space:]]*${ver//./\\.}[[:space:]]*\|)|^#+[[:space:]]+${ver//./\\.}\b" docs/CHANGELOG.md; then
    echo "    ⚠️  Header VERSION $ver ไม่พบใน CHANGELOG.md (อาจเป็น historical — ไม่ block)"
    warnings=$((warnings+1))
  fi
done

# Check 4: CHANGELOG.md "Unreleased" section handling
echo ""
echo "  📌 Check: Unreleased section (ถ้ามี)"

if grep -qE "^#+[[:space:]]+(Unreleased|UNRELEASED|\\[Unreleased\\])" docs/CHANGELOG.md; then
  unreleased_section=$(awk '/^#+[[:space:]]+(Unreleased|UNRELEASED|\[Unreleased\])/{flag=1; next} /^#+[[:space:]]/{if(flag){flag=0}} flag' docs/CHANGELOG.md)
  if [[ -n "$unreleased_section" ]]; then
    echo "    ℹ️  พบ Unreleased section — ตรวจว่า APP_VERSION ตรงกับ version ล่าสุด"

      echo "       → อาจต้องย้ายเนื้อหา Unreleased ไปยัง version $APP_VERSION"
      warnings=$((warnings+1))
    fi
  fi
fi

echo ""
echo "📊 Result → failures: $failures, warnings: $warnings"

if [[ $failures -eq 0 ]]; then
  echo "  ✅ CHANGELOG sync OK (warnings are advisory)"
  exit 0
else
  echo "  ❌ $failures sync issues found"
  exit 1
fi
